# 符号与关系方向契约

本表是全文的强制约定。所有**依赖关系**均按 **source → sink** 读取；不得把同一依赖的两个分量在不同章节交换。访问关系始终按“语句实例 → 内存位置”读取；若需反向匹配，必须显式使用逆关系。

## 基本对象

| 记号 | 定义 |
|---|---|
| `p` | 参数向量；例如 stencil 中可取 `p = (T, N)`。 |
| `C(p)` | 参数上下文，即只约束参数的公式，例如 `T ≥ 1 ∧ N ≥ 3`。它与每个语句的循环域分开书写。 |
| `S`、`T` | 语句标识；下标 `S`、`T` 不是时间变量。 |
| `x`、`y` | 语句实例（迭代向量）。在涉及两个语句时，`x` 通常属于 source 语句，`y` 属于 sink 语句。 |
| `D_S(p)` | 语句 `S` 的参数化迭代域；语句实例写作 `x ∈ D_S(p)`。完整约束可写为 `C(p) ∧ x ∈ D_S(p)`。 |
| `M` | 抽象内存位置集合；可用数组名及下标元组区分位置。 |
| `R_S ⊆ D_S × M` | 语句 `S` 的读访问关系；`(x,m) ∈ R_S` 表示实例 `x` 读取位置 `m`。 |
| `W_S ⊆ D_S × M` | 语句 `S` 的写访问关系；`(x,m) ∈ W_S` 表示实例 `x` 写入位置 `m`。 |
| `θ_S : D_S → ℤ^k` | 语句 `S` 的 `k` 维仿射调度，将实例映到整数时间戳。 |
| `Δ_{S→T} ⊆ D_S × D_T` | 从语句 `S` 到语句 `T` 的依赖；第一元是 **source** 实例，第二元是 **sink** 实例。若 `(x,y) ∈ Δ_{S→T}`，则 `x` 必须先于 `y`。 |

`D_S`、`R_S`、`W_S` 在含参数时均隐含同一 `p` 与上下文 `C(p)`；为避免把参数约束误当迭代坐标，展开公式时仍显式写出 `C(p)`。

## 集合、关系与复合

对关系 `R ⊆ X × Y` 与 `Q ⊆ Y × Z`，全文固定采用

\[
(Q \circ R)(x,z) \Longleftrightarrow \exists y:\; R(x,y) \land Q(y,z).
\]

也就是说先沿 `R` 由 `x` 到 `y`，再沿 `Q` 由 `y` 到 `z`；书写次序与函数复合一致。依赖推导中若需要沿访问关系反向匹配内存位置，必须显式写逆关系；这不会悄然翻转依赖的 source 与 sink。

| 记号 | 定义 |
|---|---|
| `R^{-1}` | 逆关系：`(y,x) ∈ R^{-1} ⇔ (x,y) ∈ R`。它只翻转该访问/映射，不改变已定义依赖 `Δ_{S→T}` 的语义。 |
| `domain(R)` | `{x \mid ∃y: (x,y) ∈ R}`。 |
| `range(R)` | `{y \mid ∃x: (x,y) ∈ R}`。 |
| `R[A]` 或 `image(R,A)` | 像：`{y \mid ∃x ∈ A: (x,y) ∈ R}`。 |
| `R^{-1}[B]` 或 `preimage(R,B)` | 逆像：`{x \mid ∃y ∈ B: (x,y) ∈ R}`。 |
| `proj_I(X)` | 投影：保留集合/关系 `X` 的坐标集合 `I`，等价于对其余坐标作存在量词消去。 |
| `lex<` | 向量的严格词典序：在第一个不同分量上左侧较小。例如 `a lex< b` 表示 `a` 严格早于 `b`。 |
| `x ≺_θ y` | 调度诱导顺序：`θ_S(x) lex< θ_T(y)`；只有在语句身份和相关参数明确时使用。 |

对依赖合法性，约定为：

\[
(x,y) \in \Delta_{S\to T} \implies \theta_S(x)\;\mathit{lex<}\;\theta_T(y).
\]

这条式子始终从 source 的调度到 sink 的调度。不要把它改写为“sink 到 source”的不等式后仍称作同一方向。

## 整数依赖与 Farkas 约束化的适用范围

依赖 `Δ_{S→T}` 的实例坐标和调度时间戳属于整数格点。普通 Farkas 引理直接处理的是有理/实多面体 `P ⊆ ℝ^d` 上仿射式的全称非负性，而不是任意整数点集上的无条件等价命题。因此，第 9 章在用 Farkas 把调度合法性有限化时必须注明所采用的路径：

1. 若对实松弛 `P` 证明非负，则该结论对 `P ∩ ℤ^d` 自动成立；这是可用于调度构造的**充分条件**，但可能排除仅在整数点上合法的候选。
2. 若需要对整数实例精确刻画，先明确用整数凸包 `conv(P ∩ ℤ^d)`（在可获得其描述时）替代 `P`，再在该凸包上应用相应的实多面体论证。

不得把上述任一路径表述为“普通 Farkas 对任意整数 Presburger 集合自动充要”。析取、同余或非凸结构还需先分支、分解或改用整数/Presburger 方法。

## 多面体、整数点与 Presburger 集合

`P ⊆ ℝ^d` 表示实数空间中的（有理）凸多面体；`P ∩ ℤ^d`（LaTeX 形式为 `P \cap \mathbb{Z}^d`）才表示它的整数格点集合。二者不是同一对象：实数多面体可含连续点，而循环实例、调度时间戳和数组下标属于整数格点。

一般 Presburger 集合允许布尔组合、量词消去后得到的析取、同余与周期结构；它不必是单个凸多面体。术语“polyhedral”在工程章节中通常指其可处理的仿射子类，不能据此抹去上述区别。

## 例：贯穿 stencil 的域和访问

令 `p = (T,N)`，`C(p) : T ≥ 1 ∧ N ≥ 3`。对唯一更新语句 `S`，

\[
D_S(p)=\{(t,i)\in\mathbb{Z}^2\mid 1\le t\le T\ \land\ 1\le i<N-1\}.
\]

其写访问可记为 `W_S(t,i) = A[t,i]`，三条读访问分别为 `A[t-1,i-1]`、`A[t-1,i]`、`A[t-1,i+1]`。这些都是从实例到内存位置的关系；在构造依赖时，生产者实例为 source，消费者实例为 sink。
